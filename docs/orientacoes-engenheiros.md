# Orientações para Engenheiros — Uso do Sistema

> Última atualização: julho 2026

Este documento reúne cuidados simples que fazem o sistema funcionar melhor. Siga essas orientações ao preparar e importar seu cronograma.

---

## 1. Sempre salve o baseline antes de importar

Antes de importar o arquivo no sistema, salve o baseline no MS Project:

**Projeto → Definir Baseline → OK**

Isso garante que o sistema saiba comparar corretamente o que foi planejado com o que está sendo realizado.

## 2. Não renomeie o projeto após a primeira importação

Depois que um projeto é importado pela primeira vez, evite mudar o nome dele no MS Project. Trocar o nome pode fazer o sistema não reconhecer que é o mesmo projeto nas importações seguintes.

## 3. Envie o arquivo original do MS Project (.mpp)

Envie sempre o arquivo `.mpp` original, do jeito que ele é salvo pelo MS Project. Não exporte para XML ou para Excel antes de importar — essas conversões podem perder informações do cronograma (como vínculos entre tarefas e a estrutura hierárquica).
